---
title: Get Neighbors
description: Retrieve immediate neighbors of a node with full details
---

## Endpoint

```
GET /graph/neighbors/:id
```

Returns the immediate neighbors (both incoming and outgoing connections) of a specific node, including full node details and edge weights.

## Path Parameters

<ParamField path="id" type="string" required>
  The UUID of the node whose neighbors to retrieve
</ParamField>

## Query Parameters

<ParamField query="user" type="string" required>
  The Last.fm username whose graph to query
</ParamField>

## Response

<ResponseField name="id" type="string">
  The UUID of the requested node
</ResponseField>

<ResponseField name="node" type="CompactGraphNode">
  Full details of the requested node (see [Get Node](/api/graph/get-node) for schema)
</ResponseField>

<ResponseField name="next" type="object">
  Outgoing neighbors: UUID → neighbor details with edge weight
  
  <Expandable title="Next neighbor structure">
    <ResponseField name="node" type="CompactGraphNode">
      Full node details with additional `id` field
      <Expandable title="Node properties">
        <ResponseField name="id" type="string">
          UUID of the neighbor node
        </ResponseField>
        <ResponseField name="songKey" type="string">
          Canonical song identifier
        </ResponseField>
        <ResponseField name="name" type="string">
          Track name
        </ResponseField>
        <ResponseField name="artists" type="string[]">
          Artist names
        </ResponseField>
        <ResponseField name="totalPlays" type="integer">
          Total plays
        </ResponseField>
        <ResponseField name="pageRank" type="number">
          PageRank score
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="weight" type="integer">
      Number of transitions from the source node to this neighbor
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="previous" type="object">
  Incoming neighbors: UUID → neighbor details with edge weight
  
  <Expandable title="Previous neighbor structure">
    <ResponseField name="node" type="CompactGraphNode">
      Full node details with additional `id` field
    </ResponseField>
    
    <ResponseField name="weight" type="integer">
      Number of transitions from this neighbor to the source node
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

This endpoint is useful for:

- Building local graph visualizations around a specific song
- Finding songs commonly played before or after a given track
- Exploring musical transitions in your listening patterns
- Identifying "hub" songs with many connections

## Notes

<Note>
- The `next` field shows songs you typically listen to **after** the requested song
- The `previous` field shows songs you typically listen to **before** the requested song
- Edge weights represent the number of times that transition occurred
- Each neighbor includes full node details, making this endpoint more expensive than [Get Node](/api/graph/get-node)
</Note>
