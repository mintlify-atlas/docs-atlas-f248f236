---
title: Get Node
description: Retrieve a single node by its UUID
---

## Endpoint

```
GET /graph/node/:id
```

Returns detailed information about a single node (song) in the listening graph.

## Path Parameters

<ParamField path="id" type="string" required>
  The UUID of the node to retrieve (URL-encoded if necessary)
</ParamField>

## Query Parameters

<ParamField query="user" type="string" required>
  The Last.fm username whose graph to query
</ParamField>

## Response

Returns a `CompactGraphNode` object (see [Get Graph](/api/graph/get-graph) for full schema).

<ResponseField name="songKey" type="string">
  Canonical song identifier in format `artist::track`
</ResponseField>

<ResponseField name="mbid" type="string">
  MusicBrainz ID from Last.fm (if available)
</ResponseField>

<ResponseField name="name" type="string">
  Track name
</ResponseField>

<ResponseField name="artists" type="string[]">
  List of artist names
</ResponseField>

<ResponseField name="albumName" type="string">
  Album name (if available)
</ResponseField>

<ResponseField name="lastfmUrl" type="string">
  Last.fm URL for the track
</ResponseField>

<ResponseField name="imageUrl" type="string">
  Album artwork URL
</ResponseField>

<ResponseField name="next" type="object">
  Outgoing edges: UUID → edge weight
</ResponseField>

<ResponseField name="previous" type="object">
  Incoming edges: UUID → edge weight
</ResponseField>

<ResponseField name="totalPlays" type="integer">
  Total number of times this song was played
</ResponseField>

<ResponseField name="sources" type="string[]">
  Data sources (currently only `["lastfm"]`)
</ResponseField>

<ResponseField name="sourcePlays" type="object">
  Breakdown of plays per source
</ResponseField>

<ResponseField name="pageRank" type="number">
  PageRank score (centrality measure)
</ResponseField>

<ResponseField name="clusterId" type="integer">
  Community cluster ID
</ResponseField>

<ResponseField name="playDates" type="string[]">
  ISO 8601 timestamps of all plays
</ResponseField>

<ResponseField name="positions" type="object">
  Pre-computed layout positions for visualization
</ResponseField>

## Notes

<Note>
- The `:id` parameter should be a valid UUID
- Node IDs are stable for a given user's graph but may change if the graph is rebuilt
- Use [Get Neighbors](/api/graph/get-neighbors) to get enriched neighbor information
</Note>
