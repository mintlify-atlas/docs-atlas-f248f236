## Overview

Spotify integration in Rotations is **currently in progress**. It provides:

- 🚧 OAuth authentication
- 🚧 Playlist syncing
- 🚧 Spotify Web Playback SDK integration
- ❌ Recently played tracks for graph (planned)
- ❌ Full listening history for graph (planned)

<Note>
  Spotify support is functional but still being refined. Some features may not work as expected.
</Note>

## Requirements

- **Spotify Premium** subscription (required for Web Playback SDK)
- **Spotify Developer App** with client ID and secret
- **OAuth redirect URI** configured in Spotify dashboard

<Warning>
  The Spotify Web Playback SDK only works with Premium accounts. Free accounts cannot use Spotify playback in Rotations.
</Warning>

## Setup

<Steps>
  <Step title="Create Spotify App">
    Visit [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)
    
    - Click **Create App**
    - Name: "Rotations" (or your choice)
    - Description: "Personal music player"
    - Redirect URI: `http://localhost:4321/api/spotify/callback`
    - API: Web API, Web Playback SDK
  </Step>
  
  <Step title="Get Credentials">
    From your app dashboard:
    
    - Copy **Client ID**
    - Click **Show Client Secret** and copy it
  </Step>
  
  <Step title="Configure Environment">
    Create `site/.env` with your credentials:
    
    ```env
    PUBLIC_SPOTIFY_CLIENT_ID=your_client_id_here
    SPOTIFY_CLIENT_SECRET=your_client_secret_here
    PUBLIC_SPOTIFY_REDIRECT_URI=http://localhost:4321/api/spotify/callback
    ```
  </Step>
  
  <Step title="Restart Dev Server">
    ```bash
    cd site
    pnpm dev
    ```
  </Step>
</Steps>

## OAuth Scopes

Rotations requests these Spotify scopes:

```typescript
const scopes = [
  "user-read-private",              // Read profile
  "user-read-email",                // Read email
  "playlist-read-private",          // Read private playlists
  "playlist-read-collaborative",    // Read collaborative playlists
  "user-read-recently-played",     // Read recent tracks (future)
  "streaming",                      // Web Playback SDK
  "user-read-playback-state",      // Read player state
  "user-modify-playback-state",    // Control playback
]
```

## Authentication Flow

<Steps>
  <Step title="Click Connect Spotify">
    In the player UI, click the Spotify status button
  </Step>
  
  <Step title="Redirect to Spotify">
    User redirected to Spotify login/authorization:
    
    ```typescript
    const authUrl = new URL("https://accounts.spotify.com/authorize")
    authUrl.searchParams.set("client_id", CLIENT_ID)
    authUrl.searchParams.set("response_type", "code")
    authUrl.searchParams.set("redirect_uri", REDIRECT_URI)
    authUrl.searchParams.set("scope", scopes.join(" "))
    
    window.location.href = authUrl.toString()
    ```
  </Step>
  
  <Step title="User Grants Permission">
    User logs in and approves requested scopes
  </Step>
  
  <Step title="Callback with Code">
    Spotify redirects back with authorization code:
    
    ```
    http://localhost:4321/api/spotify/callback?code=AQD...
    ```
  </Step>
  
  <Step title="Exchange for Token">
    Backend exchanges code for access token:
    
    ```typescript
    const tokenResponse = await fetch("https://accounts.spotify.com/api/token", {
      method: "POST",
      headers: {
        "Content-Type": "application/x-www-form-urlencoded",
        "Authorization": `Basic ${btoa(`${CLIENT_ID}:${CLIENT_SECRET}`)}`
      },
      body: new URLSearchParams({
        grant_type: "authorization_code",
        code: authCode,
        redirect_uri: REDIRECT_URI,
      }),
    })
    
    const tokens = await tokenResponse.json()
    // { access_token, refresh_token, expires_in }
    ```
  </Step>
  
  <Step title="Store Tokens">
    Tokens stored in session/localStorage for API calls
  </Step>
</Steps>

## Playlist Sync

<Note>
  Playlist sync is partially implemented. Currently in development.
</Note>

### Fetching Playlists

```typescript
// Get user's playlists
const response = await fetch("https://api.spotify.com/v1/me/playlists", {
  headers: {
    "Authorization": `Bearer ${accessToken}`
  }
})

const data = await response.json()

// data.items = SpotifyPlaylist[]
for (const playlist of data.items) {
  console.log(playlist.name, playlist.tracks.total)
}
```

### Fetching Tracks

```typescript
// Get playlist tracks
const response = await fetch(
  `https://api.spotify.com/v1/playlists/${playlistId}/tracks`,
  {
    headers: { "Authorization": `Bearer ${accessToken}` }
  }
)

const data = await response.json()

// Convert to Rotations format
const tracks: FuckingTrack[] = data.items
  .filter(item => item.track !== null)  // Skip deleted tracks
  .map(item => ({
    id: `track-spotify-${item.track.id}` as TrackId,
    name: item.track.name,
    artists: item.track.artists.map(a => a.name),
    time_ms: item.track.duration_ms,
    tags: [],  // Spotify doesn't provide genre tags per track
    audio: {
      type: "spotify",
      id: item.track.id,
    },
  }))
```

## Web Playback SDK

Rotations uses the Spotify Web Playback SDK for native playback:

```typescript
// From site/src/hooks/PlayerContext.tsx

// Initialize Spotify Player
const initSpotifyPlayer = async (accessToken: string) => {
  const player = new Spotify.Player({
    name: "Rotations Player",
    getOAuthToken: (cb) => cb(accessToken),
    volume: 1.0,
  })

  // Ready
  player.addListener("ready", ({ device_id }) => {
    console.log("Spotify player ready", device_id)
    setSpotifyDeviceId(device_id)
  })

  // Error
  player.addListener("initialization_error", ({ message }) => {
    console.error("Spotify init error:", message)
  })

  // Connect
  await player.connect()
  setSpotifyPlayer(player)
}
```

### Playback Control

```typescript
// For Spotify playlists, use Spotify's playback API
if (playlist?.source === "spotify") {
  const playlistSpotifyId = playlist.id.replace("play-spotify-", "")
  const trackSpotifyId = currentTrack.id.replace("track-spotify-", "")

  if (!spotifyDeviceId) {
    console.error("No Spotify device available")
    return
  }

  // Start playback
  await fetch("/api/spotify/play", {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      device_id: spotifyDeviceId,
      context_uri: `spotify:playlist:${playlistSpotifyId}`,
      offset: { uri: `spotify:track:${trackSpotifyId}` },
      position_ms: positionMs,
    }),
  })
}
```

### Toggle Play/Pause

```typescript
const togglePlayPause = useCallback(() => {
  if (playlist?.source === "spotify") {
    // Use Spotify player
    spotifyPlayerRef.current?.togglePlay()
    setIsPlaying(!isPlaying)
  } else {
    // Use HTML audio element
    const audio = audioRef.current
    if (audio) {
      isPlaying ? audio.pause() : audio.play()
      setIsPlaying(!isPlaying)
    }
  }
}, [isPlaying, playlist])
```

### Seek

```typescript
const handleSeek = useCallback((value: number) => {
  if (playlist?.source === "spotify") {
    // Spotify seek
    spotifyPlayerRef.current?.seek(value)
    setCurrentTimeMs(value)
  } else {
    // HTML audio seek
    const audio = audioRef.current
    if (audio) {
      audio.currentTime = value / 1000
      setCurrentTimeMs(value)
    }
  }
}, [playlist])
```

## Offline Mode

<Warning>
  Spotify tracks **cannot be cached** for offline playback. The Web Playback SDK requires an active internet connection and valid Premium subscription.
</Warning>

Spotify handles caching internally:
- Premium users get some offline capability via Spotify app
- Rotations always requires internet for Spotify playback
- Non-Spotify tracks (Bandcamp) still work offline

## Limitations

<Accordion title="Premium Required">
  The Spotify Web Playback SDK only works with Premium accounts. Free accounts will see an error when attempting to play Spotify tracks.
</Accordion>

<Accordion title="No Offline Playback">
  Spotify tracks cannot be cached locally. Always requires internet connection.
</Accordion>

<Accordion title="No Listening History Import">
  Currently can't import full listening history for graph analysis. Only recent 50 tracks accessible via API.
</Accordion>

<Accordion title="No Direct Downloads">
  Spotify Terms of Service prohibit downloading audio. Rotations uses streaming only.
</Accordion>

<Accordion title="Rate Limiting">
  Spotify API has rate limits (varies by endpoint). Large library imports may be slow.
</Accordion>

## Troubleshooting

<Accordion title="Authentication Fails">
  **Causes:**
  - Client ID/secret incorrect
  - Redirect URI mismatch
  - Missing scopes
  
  **Solutions:**
  - Verify credentials in `.env`
  - Check redirect URI in Spotify dashboard
  - Restart dev server after changing `.env`
</Accordion>

<Accordion title="Player Not Initializing">
  **Causes:**
  - Not a Premium account
  - Web Playback SDK script not loaded
  - Browser blocking third-party scripts
  
  **Solutions:**
  - Verify Premium subscription
  - Check browser console for errors
  - Disable ad blockers
  - Try different browser
</Accordion>

<Accordion title="Playback Fails">
  **Causes:**
  - Device ID not ready
  - Invalid track/playlist ID
  - Token expired
  - No internet connection
  
  **Solutions:**
  - Wait for "Spotify player ready" log
  - Verify IDs are correct
  - Re-authenticate if token expired
  - Check internet connection
</Accordion>

## API Endpoints

### Authentication

```typescript
// Start OAuth flow
GET /api/spotify/auth
// Redirects to Spotify login

// OAuth callback
GET /api/spotify/callback?code=...
// Exchanges code for token, redirects to app

// Refresh token
POST /api/spotify/refresh
// Returns new access token
```

### Playback Control

```typescript
// Start playback
PUT /api/spotify/play
{
  "device_id": "abc123",
  "context_uri": "spotify:playlist:xyz",
  "offset": { "uri": "spotify:track:track123" },
  "position_ms": 0
}

// Pause
PUT /api/spotify/pause

// Seek
PUT /api/spotify/seek?position_ms=30000
```

### Playlists

```typescript
// Get user playlists
GET /api/spotify/playlists
// Returns: SpotifyPlaylist[]

// Get playlist tracks
GET /api/spotify/playlists/:id/tracks
// Returns: FuckingTrack[]
```
