## Overview

The Rotations music player is a browser-based application built with Astro and React, featuring:

- ✅ Local-first storage with automatic persistence
- ✅ Multi-source playback (Bandcamp, Spotify, YouTube)
- ✅ Automatic offline caching
- ✅ Smart rotation management
- ✅ Rich metadata display

<Info>
  The player runs entirely in your browser using IndexedDB for storage. No account or backend required!
</Info>

## Player Context

The entire player state is managed through React Context:

```typescript
// From site/src/hooks/PlayerContext.tsx
export interface PlayerContextValue {
  // State
  playlist?: FuckingPlaylist
  tracks: FuckingTrack[]
  currentTrackIndex: number
  currentTimeMs: number
  isPlaying: boolean
  currentTrack: FuckingTrack | null
  totalDuration: number

  // Actions
  setPlaylistAndTracks: (params: SetPlaylistAndTracksParams) => void
  togglePlayPause: () => void
  handleSeek: (value: number) => void
  handleTrackSelect: (index: number) => Promise<void>
  handleNextTrack: () => Promise<void>
  handlePrevTrack: () => Promise<void>
  
  // Spotify integration
  spotifyDeviceId: string | null
  setSpotifyDeviceId: (id: string | null) => void
}

export function PlayerProvider({ children, initialPlaylist, initialTracks }) {
  const [playlist, setPlaylist] = useState(initialPlaylist)
  const [tracks, setTracks] = useState(initialTracks)
  const [currentTrackIndex, setCurrentTrackIndex] = useState(0)
  const [currentTimeMs, setCurrentTimeMs] = useState(0)
  const [isPlaying, setIsPlaying] = useState(true)
  
  // Audio element management
  const audioRef = useRef<HTMLAudioElement | null>(null)
  
  // Save player state every 5 seconds
  useEffect(() => {
    const interval = setInterval(savePlayerState, 5000)
    return () => clearInterval(interval)
  }, [currentTrackIndex])
  
  return (
    <PlayerContext.Provider value={value}>
      {children}
    </PlayerContext.Provider>
  )
}
```

<Tip>
  Use the `usePlayer()` hook anywhere in your component tree to access player state and controls
</Tip>

## Audio Playback

### Source Handling

The player handles multiple audio source types:

```typescript
// Load appropriate audio source
if (playlist?.source === "spotify") {
  // Use Spotify Web Playback SDK
  await fetch("/api/spotify/play", {
    method: "PUT",
    body: JSON.stringify({
      device_id: spotifyDeviceId,
      context_uri: `spotify:playlist:${playlistSpotifyId}`,
      offset: { uri: `spotify:track:${trackSpotifyId}` },
    }),
  })
} else if (currentTrack.audio.type === "youtube") {
  // Proxy YouTube audio
  audioUrl = `/api/youtube/stream?id=${currentTrack.audio.id}`
  audioRef.current.src = audioUrl
} else {
  // Direct stream (Bandcamp) with caching
  audioUrl = await musicCache.getOrFetch(
    currentTrack.id, 
    currentTrack.audio.url
  )
  audioRef.current.src = audioUrl
}
```

### Automatic Track Transitions

```typescript
useEffect(() => {
  const audio = audioRef.current
  if (!audio) return

  const handleEnded = () => {
    if (currentTrackIndex < tracks.length - 1) {
      setCurrentTrackIndex(currentTrackIndex + 1)
    } else {
      setIsPlaying(false)  // Stop at end of playlist
    }
  }

  audio.addEventListener("ended", handleEnded)
  return () => audio.removeEventListener("ended", handleEnded)
}, [currentTrackIndex, tracks.length])
```

## State Persistence

Player state is automatically saved:

```typescript
const savePlayerState = useCallback(() => {
  const trackId = tracks[currentTrackIndex]?.id
  if (trackId) {
    db.setPlayerState({ 
      activeTrack: trackId, 
      trackTimestamp: currentTimeMs 
    })
  }
}, [tracks, currentTrackIndex, currentTimeMs])

// Auto-save every 5 seconds
useEffect(() => {
  const interval = setInterval(savePlayerState, 5000)
  savePlayerState()  // Immediate save
  
  return () => {
    clearInterval(interval)
    savePlayerState()  // Save on unmount
  }
}, [currentTrackIndex, savePlayerState])
```

**What's saved:**
- ✅ Current playlist
- ✅ Current track
- ✅ Playback position (timestamp)
- ✅ All imported playlists and tracks

**On reload:**
- Player resumes from last position
- Last playlist automatically loaded
- No re-download needed (offline cache)

## Side Tracks

The player shows "next tracks" from other playlists:

```typescript
// From site/src/components/PlayerView.tsx
const nextTracks = useMemo(() => {
  if (!currentTrack?.next_tracks || !playlist) return []
  
  return Object.entries(currentTrack.next_tracks)
    .filter(([playlistId]) => playlistId !== playlist.id)  // Other playlists
    .map(([playlistId, trackId]) => ({
      playlist: db.getPlaylist(playlistId),
      track: db.getTrack(trackId),
    }))
}, [currentTrack, playlist])

return (
  <>
    <SideTrack track={nextTracks[0]?.track} position="left" />
    {/* Main player */}
    <SideTrack track={nextTracks[1]?.track} position="right" />
  </>
)
```

This creates **visual connections** between playlists, hinting at cross-playlist listening paths.
