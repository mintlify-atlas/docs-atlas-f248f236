## Overview

In Rotations, playlists are collections of tracks from a single source (Bandcamp album, Spotify playlist, etc.) that serve as building blocks for your rotation.

<Info>
  Every playlist you add becomes part of your rotation, contributing to your listening graph and recommendations.
</Info>

## Playlist Structure

```typescript
// From site/src/shared/types.ts
export interface FuckingPlaylist {
  id: PlaylistId                    // "play-{source}-{id}"
  track_cover_uri: string           // Album artwork URL
  name: string                      // Album/playlist name
  artists: string[]                 // Artist names
  first_track: FuckingTrack        // First track (for preview)
  totalDurationMs: number           // Total length
  source: PlaylistSource | null     // "spotify" | "youtube" | "bandcamp"
}

export interface FuckingTrack {
  id: TrackId                       // "track-{source}-{id}"
  time_ms: number                   // Track duration
  name: string                      // Track title
  artists: string[]                 // Artist names
  tags?: string[]                   // Genre/mood tags
  audio: AudioSource                // Playback source
  next_tracks?: Record<PlaylistId, TrackId>  // Cross-playlist links
}
```

## Playlist Storage

Playlists are stored locally using TinyBase:

```typescript
// From site/src/lib/store.ts
export class Database {
  getPlaylist(playlistId: PlaylistId): FuckingPlaylist | null {
    const row = store.getRow("playlists", playlistId)
    if (!row || !row.id) return null

    const firstTrack = this.getTrack(row.first_track_id as TrackId)
    if (!firstTrack) return null

    const tracks = this.getTracks(playlistId)
    const totalDurationMs = tracks.reduce((acc, track) => acc + track.time_ms, 0)

    return {
      id: row.id as PlaylistId,
      track_cover_uri: row.track_cover_uri as string,
      name: row.name as string,
      artists: JSON.parse(row.artists as string),
      first_track: firstTrack,
      totalDurationMs,
      source: row.source as PlaylistSource,
    }
  }

  getPlaylists(): FuckingPlaylist[] {
    const playlists: FuckingPlaylist[] = []
    const rowIds = store.getRowIds("playlists")

    for (const rowId of rowIds) {
      const playlist = this.getPlaylist(rowId as PlaylistId)
      if (playlist) playlists.push(playlist)
    }

    return playlists
  }
}
```

**Storage location:** Browser IndexedDB via TinyBase's localStorage persister

**Schema:**
- `playlists` table - Playlist metadata
- `tracks` table - All tracks with `playlist_id` foreign key
- Automatic JSON serialization for arrays and objects

## Viewing Playlists

### Rotation View

Click **"View full rotation"** to see all playlists:

```typescript
// From site/src/components/PlayerView.tsx
function PlaylistsContent({ onPlaylistClick }) {
  const [playlists, setPlaylists] = useState(() => db.getPlaylists())

  return (
    <div className="max-w-2xl mx-auto">
      {playlists.map((playlist) => (
        <div
          key={playlist.id}

        >
          {/* Album artwork */}
          <img
            src={playlist.track_cover_uri}
            className="w-16 h-16 object-cover rounded"

          {/* Metadata */}

            <span className="font-medium">{playlist.name}</span>
            <span className="text-sm">
              {playlist.artists[0]} · {formatDuration(playlist.totalDurationMs)}
            </span>
            
            {/* Genre tags */}
            {playlist.first_track.tags?.slice(0, 5).map((tag) => (

              </span>
            ))}
          </div>
        </div>
      ))}
    </div>
  )
}
```

**Display includes:**
- Album artwork thumbnail
- Playlist name and artist
- Total duration
- Genre tags (first 5)
- Click to switch playlists

## Track Metadata

### Tags

Tracks can have genre/mood tags for discovery:

```typescript
const track: FuckingTrack = {
  id: "track-123",
  name: "Midnight City",
  artists: ["M83"],
  tags: ["electronic", "synthwave", "upbeat", "80s"],
  // ...
}
```

Tags are:
- Extracted from Bandcamp keywords
- Fetched from Spotify track data
- Displayed in playlist view
- Used for recommendations

### Next Tracks

Tracks can link to tracks in other playlists:

```typescript
const track: FuckingTrack = {
  id: "track-123",
  name: "Song A",
  next_tracks: {
    "play-abc": "track-456",  // Next track in playlist ABC
    "play-xyz": "track-789",  // Alternative in playlist XYZ
  },
  // ...
}
```

This powers:
- **Cross-playlist navigation** - Seamless transitions
- **Listening graph edges** - Visual connections
- **Smart recommendations** - Based on actual paths

## Playlist Analytics

### Total Duration

```typescript
const totalDurationMs = tracks.reduce((acc, track) => acc + track.time_ms, 0)

function formatDuration(ms: number): string {
  const minutes = Math.floor(ms / 60000)
  const hours = Math.floor(minutes / 60)
  if (hours > 0) {
    return `${hours}h ${minutes % 60}m`
  }
  return `${minutes}m`
}
```

## Multi-Playlist Features

### Rotation Time

Player shows remaining time across all playlists:

```typescript
// From site/src/components/PlayerView.tsx
const remainingMs = useMemo(() => {
  if (!currentTrack) return 0
  
  // Remaining in current track
  const remainingInCurrentTrack = currentTrack.time_ms - currentTimeMs
  
  // Remaining tracks in playlist
  const remainingTracks = tracks.slice(currentTrackIndex + 1)
  const remainingTracksTime = remainingTracks.reduce(
    (acc, track) => acc + track.time_ms, 
    0
  )
  
  return remainingInCurrentTrack + remainingTracksTime
}, [currentTrack, currentTrackIndex, currentTimeMs, tracks])

const remainingMinutes = Math.floor(remainingMs / 60000)
```

Displays: **"42 minutes left"** in player UI

### Cross-Playlist Transitions

Side tracks show what's coming from other playlists:

```typescript
const nextTracks = useMemo(() => {
  if (!currentTrack?.next_tracks) return []
  
  return Object.entries(currentTrack.next_tracks)
    .filter(([playlistId]) => playlistId !== playlist.id)  // Exclude current
    .map(([playlistId, trackId]) => ({
      playlist: db.getPlaylist(playlistId as PlaylistId),
      track: db.getTrack(trackId as TrackId),
    }))
}, [currentTrack, playlist])
```

Creates visual hints about rotation flow.

</Tip>

<Tip>

</Tip>

</Tip>

## Troubleshooting

<Accordion title="Playlist Won't Import">
  - Check URL format matches source pattern
  - Some Bandcamp albums have CORS restrictions
  - Spotify requires authentication first
  - YouTube URLs must be direct video links
</Accordion>

<Accordion title="Missing Album Artwork">
  - Source may not provide artwork
  - Try refreshing the import
  - Check browser console for CORS errors
</Accordion>

<Accordion title="Wrong Track Order">
  - Order matches source playlist/album
  - Cannot be manually reordered currently
  - Re-import if source order changed
</Accordion>
