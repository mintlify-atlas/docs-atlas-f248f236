## Overview

The Rotations graph visualization transforms your listening history into an interactive network where you can:

- See how songs connect through listening patterns
- Discover clusters of related music
- Find paths between any two songs
- Identify your most influential tracks via PageRank

<Info>
  The graph is built from your Last.fm scrobbles, Spotify history, and local playback data.
</Info>

## Graph Structure

### Nodes and Edges

```typescript
// From graph-server/src/graph/types.ts

// Nodes = Songs
export interface GraphNode {
  name: string
  artists: string[]
  albumName?: string
  imageUrl?: string
  
  // Weighted edges
  next: Record<SongKey, number>      // A → B (weight = transition count)
  previous: Record<SongKey, number>  // B ← A (mirror of next)
  
  // Analysis
  totalPlays: number
  pageRank?: number
  clusterId?: number
  
  // Layout
  positions?: {
    pagerank?: { x: number; y: number }
    mds?: { x: number; y: number }
    "weighted-mds"?: { x: number; y: number }
  }
}
```

**Example:**
```typescript
// "Song A" played before "Song B" three times
graph.nodes["artist a::song a"].next["artist b::song b"] = 3
graph.nodes["artist b::song b"].previous["artist a::song a"] = 3
```

## Graph Loading

```typescript
// From site/src/components/graph/useGraphData.ts
export function useGraphData() {
  const [graph, setGraph] = useState<Graph | null>(null)
  const [state, setState] = useState<"loading" | "ready" | "error">("loading")
  const [error, setError] = useState<string | null>(null)

  useEffect(() => {
    // Fetch graph from server
    fetch(`http://localhost:3001/graph?user=${username}`)
      .then((res) => res.json())
      .then((data) => {
        // Convert to Sigma graph format
        const sigmaGraph = new Graph()
        
        for (const [uuid, node] of Object.entries(data.nodes)) {
          sigmaGraph.addNode(uuid, {
            x: node.positions?.pagerank?.x ?? 0,
            y: node.positions?.pagerank?.y ?? 0,
            size: Math.log(node.totalPlays + 1) * 2,
            label: node.name,
            color: getClusterColor(node.clusterId),
          })
        }
        
        // Add edges...
        setGraph(sigmaGraph)
        setState("ready")
      })
      .catch((err) => {
        setError(err.message)
        setState("error")
      })
  }, [username])

  return { graph, state, error }
}
```

## Node Visualization

### Size

Node size = `log(totalPlays + 1) * scale`

- Larger nodes = more plays
- Logarithmic scaling prevents extreme size differences
- Minimum size ensures visibility

### Color

Nodes colored by cluster:

```typescript
function getClusterColor(clusterId: number): string {
  const colors = [
    "#FF6B6B",  // Red
    "#4ECDC4",  // Teal
    "#45B7D1",  // Blue
    "#FFA07A",  // Orange
    "#98D8C8",  // Mint
    "#F7DC6F",  // Yellow
    "#BB8FCE",  // Purple
  ]
  return colors[clusterId % colors.length]
}
```

Clusters reveal:
- Genre groupings
- Artist catalog exploration
- Temporal listening periods

### Labels

Labels shown for:
- Selected node (white, large)
- Neighbor nodes (gray, medium)
- High-zoom nodes (size threshold)

```typescript
sigma.setSetting("nodeReducer", (node, data) => {
```

## Edge Visualization

### Width

Edge width proportional to weight:

```typescript
const edgeWidth = Math.sqrt(weight) * 0.5
```

### Color

Edges colored by relevance:
- **White** (`rgba(255,255,255,0.4)`) - Connected to selected node
- **Gray** (`rgba(255,255,255,0.08)`) - Background edges
- **Hidden** - Edges not connected to selected node

### Direction

Edges are directed (A → B) but rendered as lines. Direction visible in:
- Edge hover tooltip ("A → 3x → B")
- Node detail panel (incoming vs outgoing neighbors)

## Interaction Modes

### Selection Mode

```typescript
// From site/src/components/graph/GraphEvents.tsx
function GraphEvents({ onSelectNode, hiddenClusters, filter }) {
  const sigma = useSigma()

  useEffect(() => {
    const handleClickNode = (event: { node: string }) => {
      const nodeKey = event.node
      const attrs = sigma.getGraph().getNodeAttributes(nodeKey)
      
      // Gather neighbors
      const neighbors: SelectedNode["neighbors"] = []
      sigma.getGraph().forEachOutEdge(nodeKey, (edge, edgeAttrs, source, target) => {
        neighbors.push({
          key: target,
          attrs: sigma.getGraph().getNodeAttributes(target),
          weight: edgeAttrs.weight,
          direction: "outgoing",
        })
      })
      
      sigma.getGraph().forEachInEdge(nodeKey, (edge, edgeAttrs, source) => {
        neighbors.push({
          key: source,
          attrs: sigma.getGraph().getNodeAttributes(source),
          weight: edgeAttrs.weight,
          direction: "incoming",
        })
      })
      
      neighbors.sort((a, b) => b.weight - a.weight)
      
      onSelectNode({ key: nodeKey, attrs, neighbors })
    }

    sigma.on("clickNode", handleClickNode)
    return () => sigma.off("clickNode", handleClickNode)
  }, [sigma, onSelectNode])

  // Apply visual filters...
}
```

**On node click:**
1. Highlight selected node (white)
2. Highlight neighbors (gray)
3. Dim all other nodes
4. Show only connected edges
5. Display detail panel

## Search

```typescript
// From site/src/components/graph/SearchBar.tsx
export function SearchBarInner({ onSelect }) {
  const [query, setQuery] = useState("")
  const [results, setResults] = useState<SearchResult[]>([])
  const sigma = useSigma()

  const handleSearch = useCallback(
    debounce((q: string) => {
      if (!q) {
        setResults([])
        return
      }

      const graph = sigma.getGraph()
      const matches: SearchResult[] = []

      graph.forEachNode((key, attrs) => {
        const label = attrs.label.toLowerCase()
        const artist = attrs.artists?.join(" ").toLowerCase() ?? ""
        const query = q.toLowerCase()

        if (label.includes(query) || artist.includes(query)) {
          matches.push({ key, label: attrs.label, artists: attrs.artists })
        }
      })

      setResults(matches.slice(0, 50))  // Limit results
    }, 300),
    [sigma]
  )

  return (
    <div className="relative">
      <input
        value={query}
        onChange={(e) => {
          setQuery(e.target.value)
          handleSearch(e.target.value)
        }}
        placeholder="Search songs..."
        className="w-64 px-3 py-2 bg-[#1a1a1a] text-white rounded"
      />
      
      {results.length > 0 && (
        <div className="absolute top-full mt-1 w-full bg-[#1a1a1a] rounded">
          {results.map((result) => (
            <div
              key={result.key}
              onClick={() => {
                onSelect(result.key)
                setQuery("")
                setResults([])
              }}
              className="px-3 py-2 cursor-pointer hover:bg-white/5"
            >
              <p className="text-white text-sm">{result.label}</p>
              <p className="text-white/50 text-xs">{result.artists.join(", ")}</p>
            </div>
          ))}
        </div>
      )}
    </div>
  )
}
```

**Search features:**
- Fuzzy matching on song name and artist
- Real-time results (debounced)
- Click result to navigate
- Keyboard navigation (future)

## Layout Modes

The graph supports multiple layout algorithms:

### PageRank Layout (Default)

Nodes positioned radially based on PageRank score:
- High PageRank = center
- Low PageRank = periphery
- Angle = cluster ID

### MDS Layout

Multidimensional scaling based on graph distance:
- Preserves topological structure
- Similar songs positioned nearby

### Weighted MDS Layout

MDS weighted by edge strength:
- Frequently connected songs closer together
- Accounts for transition intensity

<Note>
  Layouts are pre-computed server-side and stored in the database for instant loading.
</Note>
